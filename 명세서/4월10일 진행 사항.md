## 진행 사항

### 5-A: 모델 기본 정보 — 이미 구현 완료 확인

- `Render_Model()` 내 기본 정보 표시 (Type, Meshes, Materials, Bones, Animations) 이미 적용되어 있음
- `Get_Components()`로 CModel 접근, `static_cast` 사용 확인

### 5-B: 애니메이션 컨트롤 — 구체화 + 구현 완료

#### 5-B-1: 애니메이션 리스트

- TreeNode("Animation List") 안에 Selectable로 애니메이션 목록 표시
- **인덱스 표시 포함**: `[0] Idle`, `[1] Run` 형식 — 코드에서 `Set_AnimationIndex(n)` 사용 시 참조 용도
- 현재 재생 중인 항목 강조(selected), 클릭 시 `Set_AnimationIndex` 호출로 전환

#### 5-B-2: Loop 토글

- `ImGui::Checkbox("Loop", &bLoop)` — 현재 애니메이션의 bIsLoop 런타임 변경
- `CModel::Set_AnimationLoop` 신규 구현: `m_isAnimLoop` 캐시 + `CAnimation::m_isLoop` 원본 동기화
- `CAnimation::Set_IsLoop(_bool)` 인라인 setter 추가

#### 5-B-3: 타임라인 정보

- `ImGui::ProgressBar` — TrackPosition / Duration 진행률 표시
- 오버레이 텍스트: `"%.1f / %.1f"` 형식으로 현재 위치/전체 길이 표시

## 설계 결정사항

- **애니메이션 인덱스 표시**: 리스트에 `[index] name` 형식으로 표시 — 게임플레이 코드 작성 시 인덱스 참조 용도
- **static_cast 사용**: `Com_Model` 태그로 검색된 컴포넌트는 CModel 확정이므로 dynamic_cast 불필요
- **bool 캐싱 패턴**: `Get_ModelType()` 중복 호출 대신 `bIsAnim` 변수에 캐싱
- **Set_AnimationLoop 동기화**: CModel의 `m_isAnimLoop` 캐시와 CAnimation의 `m_isLoop` 원본 둘 다 변경 — `Set_AnimationIndex`에서 캐시를 갱신하는 기존 흐름과 일관
