## 1. 사전 확인 — `CModel::Get_BoneMatrixPtr` 의미

발 위치를 모델 로컬 공간에서 비교하려면 **부모 체인이 누적된 Combined 행렬**이 필요하다 (로컬만 쓰면 Pelvis/Spine 회전이 누락되어 캐릭터가 회전해도 발 z 가 거의 안 변함).

`Engine/Private/Model.cpp:168` 확인 결과:

```cpp
const _float4x4* CModel::Get_BoneMatrixPtr(const _char* pBoneName) const
{
    _int iIndex = Get_BoneIndex(pBoneName);
    if (iIndex < 0)
        return nullptr;

    return m_Bones[iIndex]->Get_CombinedTransformMatrixPtr();
}
```

→ 이름 버전도 내부적으로 `Get_CombinedTransformMatrixPtr()` 를 호출. **이미 Combined 반환**.
→ 어제 작성한 코드블록의 `Get_BoneMatrixPtr("FX_Point_L_Foot")` 그대로 사용 가능. 별도 변경 불필요.

**호출 시점 주의**: `CModel::Play_Animation` 에서 본 로컬 → Combined 누적이 갱신되므로, `Pick_RunEndAction` 호출은 Play_Animation 이후 시점이 정확. Update 시작부에서 호출하면 1프레임 지연이 생기지만 실용상 무시 가능.

### 2.2 `Client/Private/Body_Player.cpp` — Pick_RunEndAction 신규 (Free 위 어디든)

```cpp
CHARACTER_ACTION CBody_Player::Pick_RunEndAction() const
{
    // 발 본의 모델 로컬 z 비교 — 더 작은 z (= 더 뒤) 쪽 발로 멈춤
    if (nullptr == m_pModelCom)
        return CHARACTER_ACTION::RUN_END_LEFT;

    const _float4x4* pLFoot = m_pModelCom->Get_BoneMatrixPtr("FX_Point_L_Foot");
    const _float4x4* pRFoot = m_pModelCom->Get_BoneMatrixPtr("FX_Point_R_Foot");

    if (nullptr == pLFoot || nullptr == pRFoot)
        return CHARACTER_ACTION::RUN_END_LEFT;

    // _43 = translation.z (모델 로컬 forward 축 +Z 가정)
    return (pLFoot->_43 < pRFoot->_43)
        ? CHARACTER_ACTION::RUN_END_LEFT
        : CHARACTER_ACTION::RUN_END_RIGHT;
}
```

> **주의**: 모델 forward 축이 +Z 가정. OverDrive 모델이 -Z forward 면 부등호 한 줄만 뒤집기. L/R 이 시각적으로 반대로 나오면 그 신호.

```cpp
void CPlayer_StateMachine::Update_LocoMotion(const PLAYER_INTENT_FRAME& Intent)
{
    m_bLastHasMoveIntent = Intent.bHasMoveIntent;

    // (1) DASH 입력 처리 — 우선순위 최고
    if (true == Intent.bDashRequested && nullptr != m_pOwner)
    {
        if (true == m_pOwner->Can_ConsumeDashCharge())
        {
            const CHARACTER_ACTION eDashAction = (false == Intent.bHasMoveIntent)
                ? CHARACTER_ACTION::BACK_DASH
                : CHARACTER_ACTION::DASH;

            if (CHARACTER_ACTION::DASH == eDashAction)
                m_pOwner->Face_DirectionImmediately(Intent.vMoveDirWorld);

            if (true == Try_Transition(ETOUI(eDashAction)))
                m_pOwner->Consume_DashCharge();
            return;
        }
    }

    const CHARACTER_ACTION eCurrent = Get_CurrentCharacterAction();

    // (2) RUN / RUN_FAST 중 입력 종료 → 발 위치 분기
    if (CHARACTER_ACTION::RUN == eCurrent || CHARACTER_ACTION::RUN_FAST == eCurrent)
    {
        if (false == Intent.bHasMoveIntent && nullptr != m_pOwner)
        {
            const CHARACTER_ACTION eEndAction = m_pOwner->Pick_RunEndByFoot();
            Try_Transition(ETOUI(eEndAction));
        }
        return;
    }

    // (3) RUN_END_* 재생 중 다시 이동 입력 → RUN 으로 가로채기
    if (CHARACTER_ACTION::RUN_END_LEFT == eCurrent || CHARACTER_ACTION::RUN_END_RIGHT == eCurrent)
    {
        if (true == Intent.bHasMoveIntent)
            Try_Transition(ETOUI(CHARACTER_ACTION::RUN));
        return;
    }

    // (4) IDLE 등에서 이동 입력 → RUN 직행 (WALK 안 거침)
    if (true == Intent.bHasMoveIntent)
        Try_Transition(ETOUI(CHARACTER_ACTION::RUN));
    else
        Try_Transition(ETOUI(CHARACTER_ACTION::IDLE));
}
```

void CPlayer_StateMachine::OnNotify(const NOTIFY_EVENT& Event)
{
    switch (Event.eType)
    {
    case NOTIFY_TYPE::ACTION_FINISHED:
    {
        const CHARACTER_ACTION eFinished = static_cast<CHARACTER_ACTION>(Event.iPayload);

        // DASH/BACK_DASH 종료 → 입력 있으면 RUN_FAST, 없으면 발 위치 분기
        const _bool bDashFinished =
            (CHARACTER_ACTION::DASH == eFinished) ||
            (CHARACTER_ACTION::BACK_DASH == eFinished);

        if (bDashFinished)
        {
            if (true == m_bLastHasMoveIntent)
            {
                Try_Transition(ETOUI(CHARACTER_ACTION::RUN_FAST));
            }
            else if (nullptr != m_pOwner)
            {
                Try_Transition(ETOUI(m_pOwner->Pick_RunEndByFoot()));
            }
            else
            {
                __super::On_ActionFinished();
            }
            break;
        }

        // RUN_END_LEFT / RIGHT 종료 → IDLE
        const _bool bRunEndFinished =
            (CHARACTER_ACTION::RUN_END_LEFT == eFinished) ||
            (CHARACTER_ACTION::RUN_END_RIGHT == eFinished);

        if (bRunEndFinished)
        {
            Try_Transition(ETOUI(CHARACTER_ACTION::IDLE));
            break;
        }

        // 그 외 → 기본 AutoReturn (Policy 의 eReturnAction)
        __super::On_ActionFinished();
        break;
    }

    case NOTIFY_TYPE::ANIM_EVENT:
        // Step C 이후 AnimNotify 처리 예정
        break;

    default:
        break;
    }
}

## 6. 애니메이션 cross-fade 블렌딩 인프라 도입

오후 작업 — RUN_FAST_LEFT/RIGHT lean 분기를 매끄럽게 부활시키기 위한 사전 작업으로, **모든 애니메이션 전환에 적용 가능한 cross-fade 블렌딩 시스템**을 Engine 계층에 구축. Sync marker 는 이번 범위에서 제외.

### 6.1 설계 결정

- **Pose 표현**: 본별 RTS triple (Scale vec3 + Rotation quaternion + Translation vec3) 로 평가 후 lerp/slerp
- **이중 클립 진행**: From / To 두 클립 모두 자체 시간 진행 + 자체 RM 정책. Lerp 가중치 `w = blend_elapsed / blend_duration`
- **Blend 중 재요청 정책**: 현재 lerp 결과를 새 From 으로 **정적 캡처** (bFromIsStatic=true) 후 즉시 새 To 로 재시작. From 측 RM 은 0 처리
- **Root Motion lerp**: `vDelta = (1-w)*FromDelta + w*ToDelta`. 어느 한쪽이 RM 비사용 클립이면 그쪽은 0. 부드러운 RM 감속 / 점진 가속
- **Phase 보존**: `bRestartOnEnter=false` 인 클립(locomotion 류)은 To 의 트랙 위치를 0 으로 리셋하지 않음 → 발 위상 유지
- **BlendTime SSOT**: `CHARACTER_ACTION_POLICY::fEnterBlendTime` (locomotion 0.15s, dash 0.05s)

## 7. 런타임 검증 — T13

GameApp 실행 후 발견된 이슈 및 대응:

### 7.1 첫 씬 어색함 — 해결됨
**증상**: 게임 시작 시 다른 모션이 잠깐 보이다가 IDLE 로 전환.
**원인**: 첫 `Play_Action(IDLE)` 호출 시 `m_bHasCurrentClip=false` 인데도 블렌드 분기로 진입.
**해결**: `AnimController::Play` 의 첫 재생 케이스 즉시 전환 분기 (위 6.3.3).

### 7.2 DASH 진입 시 약 0.25m forward jump — 해결됨 (Fix B)
**증상**: DASH 시작 첫 프레임에 캐릭터가 갑자기 앞으로 점프.
**원인**: blend 종료 후 정규 RM path 첫 프레임에서 `m_bRootMotionInitialized=false` → INIT 분기로 prev 캡처. 그러나 blend 동안 DASH 트랙이 이미 진행되어 있어, 다음 프레임 T_cur 와 큰 차이.
**해결** (Fix B — `Engine/Private/Model.cpp` `Play_Animation_Blended` 의 (8) 블렌드 완료 처리):

```cpp
if (w >= 1.f)
{
    m_iCurrentAnimationIndex = m_iBlendToIndex;
    m_isAnimLoop = m_Animations[m_iCurrentAnimationIndex]->Get_IsLoop();

    _bool bToUseRM = m_Animations[m_iCurrentAnimationIndex]->Get_UseRootMotion();
    _float3 vPrevTo_Backup = m_vPrevRootTranslation_To;
    _bool bInitTo_Backup = m_bBlendRMInitialized_To;
    _float3 vBind_Backup = m_vBindRootTranslation;

    Set_RootMotionEnabled(bToUseRM);  // 내부에서 Reset_RootMotionState 호출

    if (bToUseRM && bInitTo_Backup)
    {
        m_vPrevRootTranslation = vPrevTo_Backup;
        m_vBindRootTranslation = vBind_Backup;
        m_bRootMotionInitialized = true;
    }

    m_bIsBlending = false;
    m_bFromIsStatic = false;
    m_bBlendRMInitialized_From = false;
    m_bBlendRMInitialized_To = false;
}
```

blend 동안 추적해 온 To 측 prev 를 정규 path 로 핸드오버 → INIT 분기 우회.

### 7.3 RUN_END_*→IDLE 시 backward 현상 — 해결됨 (데이터 재설정)
**증상**: 달리다 멈출 때 RUN_END_* 가 슬라이드되며 멈췄는데, IDLE 로 전환되는 순간 캐릭터가 슬라이드 시작 지점 부근으로 돌아감.
**원인 추정**: SLMD 바이너리에 stale 한 RM 데이터.
**해결**: Editor 에서 해당 클립들의 RM 플래그 재설정 후 저장 → 정상 동작.
> 코드 레벨로는 `Extract_RootMotion` 에 universal bind-fix (Fix C) 를 임시 적용한 상태로 두었으나, 데이터가 정상화된 이후로는 **선택적**. 그대로 두어도 동작에 문제 없으며, bind=(0,0,0) 휴리스틱이 fragile 한 점만 인지.
