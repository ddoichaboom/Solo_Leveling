## 1. 설계 결정

### 1.1 무기 가시성 정책

- **Hidden 모드** (기본 상태): IDLE/WALK/RUN/RUN_FAST*/RUN_END*/DASH/BACK_DASH 등 비전투 액션
- **Visible 모드**: 공격/스킬 액션 진행 중 + 액션 종료 후 IDLE 머무는 동안 (3초 한정)
- **전환 트리거 4종**:
  - Combat/Skill 입력 → 즉시 Visible (Draw 애니메이션 없음, 그냥 등장)
  - Visible + IDLE 3초 경과 → UNDRAW 액션 재생 → 종료 시점에 Hidden
  - Visible 중 이동/Dash 진입 → 즉시 Hidden + Undraw 스킵
  - UNDRAW 도중 이동 입력 → 강제 캔슬 + Hidden + RUN

### 1.2 좌/우 무기 결정 규칙

| 장착 상태 | 오른손 | 왼손 |
|---|---|---|
| NONE (DEFAULT) | KnightKiller | KasakaVenomFang |
| KnightKiller (DAGGER) | KnightKiller | KasakaVenomFang |
| KasakaVenomFang (DAGGER) | KasakaVenomFang | KnightKiller |
| 그 외 DAGGER (X) | X | 단검 풀의 나머지 |
| 비-DAGGER | 해당 무기 | (없음 — 왼손 visible=false) |

비-DAGGER 무기는 왼손 비워두면 애니메이션이 알아서 왼손 위치를 따라가도록 처리.

### 1.3 CHARACTER_STATE vs Weapon Visibility

두 개념은 **독립된 축**.

- `CHARACTER_STATE` = 액션 락 여부 (LOCOMOTION = 자유 이동, COMBAT = 공격 모션 락)
- 무기 Visibility = 메시 렌더 여부

LOCOMOTION + Visible 조합 가능 (공격 직후 IDLE에서 무기 든 채 가만히 있는 시점 = 3초 타이머 가동 구간).

### 1.4 KnightKiller 명시 장착 vs DEFAULT

외형은 동일하지만:
- DEFAULT (`EQUIPPED_WEAPON_ID::NONE`) → 무기 스킬 사용 **불가**
- KnightKiller 명시 장착 → 무기 스킬 사용 **가능**

`CPlayer::Can_UseWeaponSkill()` 헬퍼로 체크 (`m_eEquippedWeapon != NONE`).

## 5. 무기 PreTransform 결정

질문: 플레이어 모델은 PreTransform `(0, 180, 0)` 적용. 무기도 동일하게 줘야 하는가?

**결론: 아니요, 동일할 필요 없음.**

- 캐릭터 PreTransform: 모델의 forward 축을 월드 forward 와 정렬
- 무기 PreTransform: 무기의 grip/blade 방향을 **소켓 본의 기대 좌표계**와 정렬
- 소켓 본은 캐릭터 회전 + 손목 자세를 이미 누적해놓은 상태이므로, 무기는 위에 얹히기만 하면 됨

KnightKiller / KasakaVenomFang 둘 다 `(0, 0, 0)` PreTransform 으로 시각적 문제 없이 부착됨. 새 무기 추가 시 시각 검증 후 결정.
