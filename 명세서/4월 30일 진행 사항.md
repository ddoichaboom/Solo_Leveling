## 3. 핫픽스 — Camera_Free / Camera_Follow VIEW 충돌

### 3.1 증상

F5 로 Play 모드 진입 후 마우스로 카메라 회전해도 WASD 가 카메라 방향이 아닌 **Camera_Free 의 정적 시점** 기준으로 동작.

### 3.2 원인

Update_Engine 의 페이즈 순서:

1. Priority_Update (전체)
2. Update (전체)
3. PipeLine 캐싱 (View Inverse, CamPosition)
4. Late_Update (전체)

VIEW 매트릭스 작성 위치:

- `CCamera::Update` → `Update_PipeLine` → VIEW 작성 (베이스)
- `CCamera_Follow::Update` — 비어 있음 (베이스 호출 X) → VIEW 미작성
- `CCamera_Follow::Late_Update` — frozen 가드 후 SpringArm 기반 VIEW 작성
- `CCamera_Free::Update` — `__super::Update` 호출 → VIEW 작성 ← **문제**

`CCamera_Free::Priority_Update` 와 `Late_Update` 에는 `Is_GameLogic_Frozen()` 게이트가 적용되어 있었는데 **`Update` 에는 누락**. 그래서 Play 모드에서도 매 프레임 Camera_Free 시점으로 VIEW 가 덮어써졌고, Player::Update 가 이 VIEW 를 읽어 `Query_CameraYaw()` 가 Camera_Free 기준 yaw 를 반환 → WASD 방향 어긋남.

렌더 시점에는 Late_Update 의 Camera_Follow 가 VIEW 를 다시 덮어쓰므로 **화면은 정상**, **입력 방향만 어긋나는** 미묘한 버그였음.

### 3.3 수정

```cpp
// Client/Private/Camera_Free.cpp
void CCamera_Free::Update(_float fTimeDelta)
{
    if (false == m_pGameInstance->Is_GameLogic_Frozen())
        return;

    __super::Update(fTimeDelta);
}
```

Priority_Update / Late_Update 와 동일한 frozen 게이트 추가. 이제:

- Edit 모드: Camera_Free 가 VIEW 작성 (Update_PipeLine 정상 호출)
- Play 모드: Camera_Free 는 모든 페이즈에서 VIEW 미터치, Camera_Follow::Late_Update 만 작성

### 3.4 검증

- F5 로 Play 진입 → 마우스로 카메라 회전 → WASD 가 카메라 Look 기준으로 정상 동작 ✓
- Style C (플레이어가 카메라 방향으로 회전하며 이동) 작동 확인 ✓

### 4.3 변경 사항

**(1) `CBody_Player`** — preview 멤버/메서드 전체 제거

- `Begin_Preview`, `Restart_Preview`, `End_Preview`, `Is_Previewing`, `Get_PreviewAnimationIndex` 제거
- 멤버 `m_bPreviewMode`, `m_iPreviewAnimationIndex` 제거
- `Update` 의 게이트를 `Is_GameLogic_Frozen()` 으로 치환

```cpp
void CBody_Player::Update(_float fTimeDelta)
{
    if (m_pGameInstance->Is_GameLogic_Frozen())
    {
        if (nullptr != m_pModelCom)
            m_pModelCom->Play_Animation(fTimeDelta);
        return;
    }

    _bool bFinished = m_pAnimController->Update(fTimeDelta);

    if (bFinished && nullptr != m_pListener) {
        NOTIFY_EVENT Event{};
        Event.eType = NOTIFY_TYPE::ACTION_FINISHED;
        Event.iPayload = ETOUI(m_eCurrentAction);
        Event.pData = nullptr;
        m_pListener->OnNotify(Event);
    }
}
```
