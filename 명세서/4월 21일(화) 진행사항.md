# 4월 21일(화) 진행사항 — Client 런타임 테스트베드 구성

Step 3 Player 입력/상태머신 설계 전에, **GameApp(런타임)** 쪽에서 맵 + 플레이어 스폰 + 이동/Root Motion 검증이 가능하도록 Client 게임 로직 경로를 먼저 정리한다. Editor는 이미 동일 구성이 올라가 있으므로 **Client Loader / Level_GamePlay에 동일 구성을 이식**하는 것이 핵심.

> Codex 협업 지침 + 현재 Solo_Leveling 코드 상태 교차 검증 결과를 기반으로 작성.

## 1. 목적과 범위

### 1차 검증 목표

- GameApp 진입 → GAMEPLAY 레벨 로드 시 **맵이 렌더링됨**
- **Player가 지정한 스폰 위치**에 등장함
- W/A/S/D 입력으로 Transform 이동 테스트 가능 (임시 단계)
- 기존 스모크 액션(Dash / BackDash / BasicAttack_01)이 **런타임에서 실행**됨
- Root Motion이 있는 클립은 **Player 월드 위치에 누적** 반영됨

### 이번 범위에서 **제외** (Step 3 이후)

- Player 입력→Intent→Action 해석 레이어 완성형 (키 매핑 테이블은 `통합_구현계획_v2.md §4.1` 참조)
- 무기 스왑(C) 완성형 / Current/Pending WeaponState 분리
- HUD / 폰트 / 2D UI
- Lock-on / 추적 카메라
- NavMesh / 지면 샘플링
- Weapon visual toggle 타이밍

### 핵심 원칙

- **Terrain은 gameplay 런타임 경로에서 완전히 제거**. 대신 `CMapObject`(ContainerObject) + `CMapStaticObject`(PartObject) 구성을 그대로 사용
- Editor 전용 경로 복붙이 아닌, **런타임에 꼭 필요한 프로토타입만** 선별 이식
- Camera_Free는 "임시 테스트 카메라"로만 유지 (주석/명세서에 명시)

### 3.3 Player 스폰 Desc 반영 (가장 중요)

대상: `Client/Public/Player.h`, `Client/Private/Player.cpp`

**현재 버그 위치**: Player.cpp:22~29
```cpp
HRESULT CPlayer::Initialize(void* pArg)
{
    GAMEOBJECT_DESC Desc{};                  // ← pArg 무시
    Desc.fSpeedPerSec = 10.f;
    Desc.fRotationPerSec = XMConvertToRadians(180.f);
    if (FAILED(__super::Initialize(&Desc)))  // ← 로컬 Desc로 덮어씀
        return E_FAIL;
    ...
}
```

**방식 선택**: `PLAYER_DESC` 신설 (권장 — Codex 지침과 동일)

이유:
- `TRANSFORM_DESC`(base) 에 이미 `vPosition / vRotationDeg / vScale / fSpeedPerSec / fRotationPerSec` 가 있어 필드는 충분하지만,
- 플레이어 고유 파라미터(초기 WeaponState, AnimSet 등)가 향후 추가될 여지가 있으므로 **전용 DESC 타입 분리**가 확장성에 유리

**설계 스케치 (Player.h)**

```
class CLIENT_DLL CPlayer final : public CContainerObject
{
public:
    typedef struct tagPlayerDesc : public CGameObject::GAMEOBJECT_DESC
    {
        // (향후) 초기 WeaponState, AnimSet 등
    } PLAYER_DESC;
    ...
};
```

**Player.cpp::Initialize(void* pArg)**

```
PLAYER_DESC Desc{};
if (nullptr != pArg)
    Desc = *static_cast<PLAYER_DESC*>(pArg);

// 기본값 보강 (caller가 0으로 뒀을 때만)
if (0.f == Desc.fSpeedPerSec)      Desc.fSpeedPerSec = 10.f;
if (0.f == Desc.fRotationPerSec)   Desc.fRotationPerSec = XMConvertToRadians(180.f);

if (FAILED(__super::Initialize(&Desc))) return E_FAIL;
```

> `__super::Initialize`(CGameObject → CContainerObject) 가 `TRANSFORM_DESC`의 `vPosition / vRotationDeg / vScale`를 반영하는지 Player.cpp 수정 전에 **반드시 직접 확인**. 만약 반영되지 않으면, Initialize 내부에서 `m_pTransformCom->Set_Position(Desc.vPosition)` 등 수동 반영 필요.

### 3.4 초기 스폰 위치 (하드코딩)

- 맵: `Hapjung_Station_1F` 원점 근처 평탄 지점
- **1차 값 제안**: `vPosition = (0.f, 1.f, 0.f)` (지면 관통 방지용 +1)
- 필요 시 맵 렌더링 확인 후 실측으로 조정. 이 단계에서 NavMesh/샘플링은 금지

**Level_GamePlay.cpp::Ready_Layer_Player 예**

```
CPlayer::PLAYER_DESC Desc{};
Desc.vPosition       = _float3(0.f, 1.f, 0.f);
Desc.vRotationDeg    = _float3(0.f, 0.f, 0.f);
Desc.vScale          = _float3(1.f, 1.f, 1.f);
Desc.fSpeedPerSec    = 10.f;
Desc.fRotationPerSec = XMConvertToRadians(180.f);

return m_pGameInstance->Add_GameObject(
    ETOUI(LEVEL::GAMEPLAY), TEXT("Prototype_GameObject_Player"),

```

## 4. 검증 체크리스트

런타임(GameApp) 실행 후:

- [ ] GAMEPLAY 진입 시 **맵이 화면에 보임** (Hapjung_Station_1F)
- [ ] **Player 메시가 보임** (스폰 위치 지면 위)
- [ ] Camera_Free RMB + WASD 로 시점 이동 동작
- [ ] **CPlayer가 WASD를 읽어** Transform 이동 (임시 블록)
- [ ] Body_Player의 `KeyState('1')` → Dash 클립 재생 (스모크 잔존)
- [ ] Body_Player의 `KeyState('2')` → BackDash 클립 재생
