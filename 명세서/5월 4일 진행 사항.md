## 1. 회전 보간 정책 — 각도 의존 RotationPerSec

### 배경

기존 `Player::PLAYER_DESC::fRotationPerSec = 1080°/s` 는 1초에 3바퀴로, 90° 회전이 약 5프레임 만에 끝나 RUN_FAST_LEFT/RIGHT 의 lean 애니메이션이 시각적으로 등록될 시간이 없었다. 그렇다고 단순히 값을 540°/s 로 낮추면 W↔S 같은 180° 뒤돌기가 호를 그리며 돌아 "스냅 뒤돌기" 체감이 사라진다.

### 결정 — 각도 의존 보간

`Player.cpp` 의 LocoMotion 회전 호출 직전에 현재 Look 과 목표 MoveDir 사이 각도를 계산하고, 각도에 따라 회전 속도를 보간한다.

| 각도 차이 | 회전 속도 | 의도 |
|---|---|---|
| ≤ 30° (`fSlowDeg`) | 540°/s (`fSlowRotDeg`) | lean 충분히 가시 |
| 30° ~ 135° (`fSnapDeg`) | 540 → 1440 선형 보간 | 자연스러운 그라데이션 |
| ≥ 135° | 1440°/s (Desc 최대값) | 거의 즉시 스냅 (180° 약 0.125s) |

### 변경 파일

- `Client/Private/Level_GamePlay.cpp:136` — `fRotationPerSec = 1440°/s` (최대치/스냅 속도)
- `Client/Private/Player.cpp` — LocoMotion 업데이트 내 회전 호출을 각도 의존 보간으로 교체. 상수 `fSlowDeg=30`, `fSnapDeg=135`, `fSlowRotDeg=540` 은 현재 const, 추후 `PLAYER_DESC` 외부화 검토.

### 체감 결과

- WASD 만으로 한 바퀴 돌 때 코너마다 lean 짧게 보임
- W → S 뒤돌기는 거의 즉시 (스냅 유지)
- 카메라 회전 시 lean 충분히 유지

## 2. RUN_FAST Variant Hysteresis

### 배경

이전 구현은 `Pick_RunFastVariant()` 가 단일 임계 (cos 0.9, ≈25.8°) 만 사용 → 경계에서 LEFT/RIGHT 진입과 FAST 복귀가 토글링 발생. 디버그 로그상 LEFT/RIGHT 진입 직후 (±45°) 단 1~2 프레임 만에 FAST (±9°) 로 복귀하는 패턴 다수 관측.

### 결정 — 진입/이탈 임계 분리 (Hysteresis)

`Pick_RunFastVariant(vMoveDirWorld, eCurrent)` 시그니처에 현재 액션을 추가하고, 현재 lean 중인지 여부에 따라 임계 분리:

| 현재 액션 | 임계값 | 의미 |
|---|---|---|
| RUN_FAST | `fEnterCos = 0.866` (30°) | 30° 넘어가면 LEFT/RIGHT 진입 |
| RUN_FAST_LEFT/RIGHT | `fExitCos = 0.966` (15°) | 15° 안으로 들어와야 FAST 복귀 |

→ ±15°~±30° 구간에서는 현재 상태 유지, 토글링 방지.

### 변경 파일

- `Client/Public/Player.h` — `Pick_RunFastVariant` 시그니처 변경 (`eCurrent` 파라미터 추가)
- `Client/Private/Player.cpp:205` — Hysteresis 로직 적용
- `Client/Private/Player_StateMachine.cpp:84` — 호출부 인자 추가

## 4. SpeedCoeff 정책 명시화 (#24)

### 배경

기존 `Player_StateMachine::On_Transition` 의 switch 는 WALK/RUN/RUN_FAST_*/IDLE/RUN_END_* 만 case 처리하고 DASH/BACK_DASH/UNDRAW 는 default 로 빠져 **이전 잔존값이 남는 silent path** 가 존재했다. 현재는 LocoMotion 가드(`bIsLocomotion` 체크)로 미사용이지만, 향후 누군가 가드를 풀거나 root motion 안 쓰는 dash 변종 추가 시 silent bug 위험.

### 결정 — 모든 액션 명시 + default 0

| 액션 | SpeedCoeff |
|---|---|
| WALK | 1.0 |
| RUN | 1.5 |
| RUN_FAST / LEFT / RIGHT | 2.0 |
| IDLE / RUN_END / RUN_END_LEFT / RIGHT | 0.0 |
| **DASH / BACK_DASH** | **0.0** (명시) — root motion 기반 이동 |
| **UNDRAW** | **0.0** (명시) |
| **default** | **0.0** (방어적) — 새 액션 추가 시 case 빠뜨려도 silent bug 대신 멈춤 |

### 변경 파일

- `Client/Public/Player.h:94` — `m_fSpeedCoeff = { 0.f }` (기존 1.f → 0.f, IDLE 기본값과 정합)
- `Client/Private/Player_StateMachine.cpp:189-210` — switch 에 DASH/BACK_DASH/UNDRAW case 추가, default 0 추가

### 검증 결과

런타임 디버그 로그(`[Trans] eFrom -> eTo | prevCoeff=X.XX → newCoeff=Y.YY`) 로 모든 시나리오 통과 확인:

- 시작 시 prevCoeff=0.00 (이전엔 1.00 잔존)
- RUN → DASH 시 newCoeff=0.00 (이전엔 1.50 잔존)
- RUN_FAST → DASH 시 newCoeff=0.00 (이전엔 2.00 잔존)
- BACK_DASH/UNDRAW 양방향 모두 0.00 일관

## 5. Editor Animation Preview 호환성 (#25)

### 문제

기존 Hierarchy 패널은 Layer 의 직속 GameObject 만 노출 → ContainerObject(`CPlayer`) 클릭 시 `m_pSelectedObject = CPlayer*` 로 잡히지만, CPlayer 에는 `CModel` 컴포넌트가 없음 (자식 PartObject `Body_Player` 가 보유). 결과적으로 Inspector 의 Animation UI 가 표시되지 않아 **Player 캐릭터의 애니메이션 프리뷰가 Editor 에서 불가능**한 상태.

### 결정 — Hierarchy 트리에 PartObject 자식 노드 노출

`Panel_Hierarchy::Render()` 의 GameObject 루프를 `Selectable` 에서 `TreeNodeEx` 로 교체. ContainerObject 인 경우 `Get_PartObjects()` 순회로 자식 PartObject 들을 들여쓰기 노드로 렌더.

```
Layer_Player
  └─ ▼ Player              ← Container, 펼침/접힘 가능
       ├─ Body_Player          ← 클릭 → Inspector Animation UI 표시
       ├─ Weapon_Right
       └─ Weapon_Left
```

### 부수 이슈 처리

| 이슈 | 처리 |
|---|---|
| **PartObject 의 DnD source** | TreePush 자식 루프 안에 `BeginDragDropSource` 호출 없음 → 자동 차단 |
| **Move/Reorder target** | 동일하게 `BeginDragDropTarget` 호출 없음 |
| **우클릭 Delete 메뉴** | `BeginPopupContextItem` 호출 없음 |
| **Delete 키 (전역)** | 기존 `for (Pair : *pLayers) find(List, pSelected)` 로직이 PartObject 를 찾지 못해 자동 차단 (변경 불필요) |
| **ImGui ID 충돌** | 외부 `PushID(iIndex)` + 내부 `PushID(pPart 포인터)` 분리 |
| **자식 없는 GameObject** | `Leaf | NoTreePushOnOpen` 플래그로 화살표 미표시, 일반 항목처럼 동작 |
| **TreeNodeEx 클릭 vs 토글** | `IsItemClicked() && !IsItemToggledOpen()` 가드로 화살표 클릭은 선택 안 됨 |

### 변경 파일

- `Editor/Private/Panel_Hierarchy.cpp` — include 2줄 추가 (`ContainerObject.h`, `PartObject.h`), `Render()` 의 오브젝트 루프 (113~173 라인) 교체

### 검증 시나리오 (전부 통과)

1. Player 옆 ▶ 화살표 클릭 → 펼쳐짐, Body_Player/Weapon_* 노출
2. Player 라벨 클릭 → Player 선택 (Inspector — Transform 만)
3. Body_Player 클릭 → Body_Player 선택 → **Inspector 에 Animation UI 표시** ✅
4. Body_Player 우클릭 → 컨텍스트 메뉴 안 뜸
5. Body_Player 드래그 시도 → 드래그 안 됨
6. Body_Player 선택 후 Delete 키 → 무반응
7. Player 자체 Move/Reorder/Delete → 정상 동작 (기존과 동일)
8. Player 더블클릭 → 펼침/접힘 토글, 선택 동시 진행 안 함
