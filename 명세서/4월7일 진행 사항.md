### Step 3: Channel.cpp Update_TransformationMatrix를 수업 코드 방식으로 변경

#### 변경 사항 (Solo_Leveling 기존 → 수업 코드 방식)
1. **결과 변수 타입**: `_float3/4` → `_vector` (Store→Load 왕복 제거, SIMD 효율)
2. **Translation W=1**: 보간 분기에서 미리 `XMVectorSetW(..., 1.f)` 설정
3. **브래킷 탐색**: `while (idx < N-1) { if (...) break; }` → `while (track >= keyFrames[idx+1].pos) ++idx;` (수업 코드 방식, boundary 체크 없이 가드 ②에 의존)
4. **루프 리셋 가드 추가**: `if (0.f == fCurrentTrackPosition) *pCurrentKeyFrameIndex = 0;`
5. **가드 ③ 유지**: `else if (track <= front().fTrackPosition) → 첫 값 고정`

### Step 5: Animation.cpp 구현

#### 핵심 구현
```cpp
HRESULT CAnimation::Initialize(const ANIMATION_DESC& Desc)
{
    strcpy_s(m_szName, Desc.szName);
    m_fDuration         = Desc.fDuration;
    m_fTickPerSecond    = Desc.fTickPerSecond;
    m_iNumChannels      = Desc.iNumChannels;

    m_Channels.reserve(m_iNumChannels);

    for (size_t i = 0; i < m_iNumChannels; i++)
    {
        CChannel* pChannel = CChannel::Create(Desc.pChannels[i]);
        if (nullptr == pChannel)
            return E_FAIL;

        m_Channels.push_back(pChannel);
    }

    m_CurrentKeyFrameIndices.resize(m_iNumChannels);

    return S_OK;
}

_bool CAnimation::Update_TransformationMatrix(const vector<CBone*>& Bones, _float fTimeDelta, _bool isLoop)
{
    m_fCurrentTrackPosition += m_fTickPerSecond * fTimeDelta;

    if (m_fCurrentTrackPosition >= m_fDuration)
    {
        if (false == isLoop) return true;
        m_fCurrentTrackPosition = 0.f;
    }

    _uint iChannelIndex = {};
    for (auto& pChannel : m_Channels)
        pChannel->Update_TransformationMatrix(Bones, m_fCurrentTrackPosition, &m_CurrentKeyFrameIndices[iChannelIndex++]);

    return false;
}
```

#### Animation `= 0.f` ↔ Channel `if (0.f == ...)` 가드의 정합성
- 루프 리셋 직후 Channel이 트랙 0을 받음 → Channel-side 가드가 keyFrameIndex를 0으로 리셋
- **두 코드가 짝으로 정확히 동작**. 한 프레임만 가드 발동, 다음 프레임부터는 정상 보간 분기

#### 대안 (선택, 미적용)
- `m_fCurrentTrackPosition -= m_fDuration` 방식 — 트랙 잔량 보존, track > 0 항상 보장. 이 경우 Channel의 `if (0.f == ...)` 가드는 죽은 코드가 됨
- 더 정밀하지만 현재 `= 0.f` 방식도 정상 동작이라 변경하지 않음

## Channel.cpp 가드 4개 의미 정리 (논의 결과)

| 가드 | 발동 조건 | 의도 |
|------|-----------|------|
| ① `iNumKeyFrames == 0` | 빈 채널 | 안전장치. division by zero / out-of-bounds 방지 |
| ② `track >= back().fTrackPosition` | 마지막 키프레임 시간 초과 | 외삽 방지. 마지막 값 고정. 비루프 종료 시 자연스러운 마지막 포즈 유지 |
| ③ `track <= front().fTrackPosition` | 첫 키프레임 시간 미만 | 역방향 외삽 방지. 첫 값 고정. 첫 키프레임이 시간 0이 아닌 경우(드묾) 또는 정확히 일치 |
| ④ `else` | 두 키프레임 사이 | 정상 보간. 브래킷 탐색 + Lerp/Slerp |

### 가드 ③ 관련 논의 결과
- 사용자 우려: "가드 ③ 살리면 첫 키프레임 이전 구간에서 보간 안 됨"
- **사실 확인**: 정확함. `[0, front().fTrackPosition]` 구간 내내 첫 값 고정
- **하지만**: 현재 Animation의 트랙 누적 방식(`+= tickPerSecond * dt` 후 사용)이라 **track > 0이 항상 보장됨** (예외: 루프 wraparound 한 프레임)
  - 첫 키프레임이 시간 0 (Assimp 일반적 케이스)인 경우: 가드 ③이 실제로 발동하지 않음 → 정지 구간 0
  - 첫 키프레임이 시간 > 0 (드묾)인 경우: 정지 구간 발생
- **결론**: 가드 ③ 유지가 표준이고 안전. 만약 첫 키프레임 ≠ 0인 데이터가 발견되면 **Editor 변환 단계에서 시간 0으로 정규화**가 진짜 해법
- 가드 ③ 제거 시 음수 비율 보간 → Scale 음수, Translation 반대 방향, Slerp 정의되지 않음 → **시각적 깨짐**

## 트러블슈팅

### 1. Channel.cpp `vTranslation` 콤마 typo (사용자 작성 단계)

#### 증상
보간 분기의 마지막 줄:
```cpp
vTranslation, XMVectorLerp(vSrcTranslation, vDstTranslation, fRatio);
```

#### 원인
`=`가 아닌 **콤마 연산자 표현식**. `vTranslation` 평가(부수효과 0) → Lerp 결과 평가(버려짐). 결과적으로 vTranslation은 `_vector{}` (0,0,0,0) 그대로.

#### 잠재적 시각 증상
보간 분기 진입 시 모든 본의 로컬 translation이 (0,0,0)으로 덮어씌워짐 → 캐릭터가 한 점으로 찌부러진 채 애니메이션 재생.

#### 해결
```cpp
vTranslation = XMVectorLerp(vSrcTranslation, vDstTranslation, fRatio);
```

### 2. Animation.cpp 복사 생성자 brace-init 함정 (사용자 작성 단계)

#### 증상
```cpp
, m_CurrentKeyFrameIndices { Prototype.m_iNumChannels }
```

#### 원인
C++ vector의 brace-init은 `initializer_list<T>` 생성자를 우선 매칭. `{N}` (size_t 한 개)을 넘기면:
- 의도: size N짜리 vector 생성
- 실제: **size 1짜리 vector, 첫 원소가 N**

#### 잠재적 런타임 증상
m_iNumChannels = 70 → m_CurrentKeyFrameIndices는 size 1, 첫 원소 70.
Update에서 `&m_CurrentKeyFrameIndices[iChannelIndex++]` 인덱싱 → out-of-bounds, 메모리 손상, 즉시 크래시.

#### 해결 (사용자가 다른 방식으로 수정)
```cpp
, m_CurrentKeyFrameIndices { Prototype.m_CurrentKeyFrameIndices }
```
**vector → vector 복사**는 brace-init이라도 `vector(const vector&)` 복사 생성자가 우선 매칭됨. 정상 동작. 이전 `{m_iNumChannels}`와 다른 패턴이 됨.

(대안: `m_CurrentKeyFrameIndices(Prototype.m_iNumChannels, 0)` 소괄호 사용)

### 3. Animation.cpp Create / Clone / Free 구현 누락 (사용자 작성 단계)

#### 증상
- `Create`: `return nullptr;` 한 줄만
- `Clone`: 본문 비어있음, return 없음
- `Free`: 본문 비어있음

#### 잠재적 영향
- Create: 모든 Animation 생성 호출이 nullptr 반환 → Animation 자체가 작동 안 함
- Clone: 정의되지 않은 동작 (스택 쓰레기 반환), C4715 경고
- Free: 채널 RefCount 감소 안 함 → 모든 채널 영구 누수

#### 해결
표준 패턴으로 모두 구현:
```cpp
CAnimation* CAnimation::Create(const ANIMATION_DESC& Desc)
{
    CAnimation* pInstance = new CAnimation();
    if (FAILED(pInstance->Initialize(Desc)))
    {
        MSG_BOX("Failed to Created : CAnimation");
        Safe_Release(pInstance);
    }
    return pInstance;
}

CAnimation* CAnimation::Clone() { return new CAnimation(*this); }

void CAnimation::Free()
{
    __super::Free();
    for (auto& pChannel : m_Channels)
        Safe_Release(pChannel);
    m_Channels.clear();
}
```

### 3. Step 1 — Solo Leveling Model Data (.bin) 포맷 v1 확정

#### 결정 사항 (5개 논의 항목)

| 항목 | 결정 | 근거 |
|---|---|---|
| 확장자 | `.bin` | 수업 코드 관행, 단순 |
| 저장 위치 | FBX 원본 폴더 (`Resources/Models/<name>/<name>.bin`) | 텍스처와 같은 폴더 → 상대 경로 단순 |
| 문자열 저장 | 고정 길이 `char[MAX_PATH]` | DESC 메모리 레이아웃과 일치, 디버깅 편의 |
| 텍스처 경로 | **Resources/ 루트 기준 상대 경로** | CLAUDE.md 의 `../../Resources/...` 패턴과 일치 |
| Magic | `"SLMD"` + version=1 | Solo Leveling Model Data |

#### 포맷 스펙 (Model.cpp 상단에 코멘트로 박제)

```
[Header]  (80 byte)
  char        magic[4]              = "SLMD"
  _uint       version               = 1
  _uint       modelType             (0=NONANIM, 1=ANIM)
  _uint       reserved              = 0
  _float4x4   PreTransformMatrix

[Meshes]  _uint iNumMeshes + MESH_DESC[]
  char        szName[MAX_PATH]
  _uint       iMaterialIndex
  _uint       iVertexStride         (sizeof(VTXMESH) or VTXANIMMESH)
  _uint       iNumVertices
  byte        vertices[iNumVertices * iVertexStride]
  _uint       iNumIndices
  _uint       indices[iNumIndices]
  // ANIM 전용:
  _uint       iNumBones (mesh-local)
  _uint       boneIndices[iNumBones]
  _float4x4   offsetMatrices[iNumBones]

[Materials]  _uint iNumMaterials + MATERIAL_DESC[]
  _uint       iNumTextures
  for each tex: _uint eType + wchar_t szPath[MAX_PATH]

[Bones]  _uint iNumBones + BONE_DESC[]   (NONANIM 이면 0)
  char        szName[MAX_PATH]
  _int        iParentIndex
  _float4x4   TransformationMatrix

[Animations]  _uint iNumAnimations + ANIMATION_DESC[] (NONANIM 이면 0)
  char        szName[MAX_PATH]
  _float      fDuration
  _float      fTickPerSecond
  _uint       bIsLoop (0/1)
  _uint       iNumChannels
  for each ch: _uint iBoneIndex + _uint iNumKeyFrames + KEYFRAME[iNumKeyFrames]
```

**KEYFRAME 은 POD** 이므로 채널당 `fread(..., sizeof(KEYFRAME), N, fp)` 한 번에 읽음 → 파일 용량의 대부분을 차지하는 섹션이 가장 빠른 경로.

### 4. Step 1 Engine .bin 로더 구현 — 진행 중 (마무리 직전)

#### 적용 대상 파일

- `Engine/Private/Model.cpp` — 포맷 상수 + Load_Binary_Desc + Free_Binary_Desc + 새 Create 오버로드

#### 주요 설계 포인트
- `static HRESULT Load_Binary_Desc(const _tchar* pBinaryPath, MODEL eType, MODEL_DESC* pOutDesc)` — .bin 을 읽어 임시 MODEL_DESC 힙 생성

- 새 Create 오버로드 `Create(device, context, eType, const _tchar* pBinaryPath)`:
  1. Load_Binary_Desc → 임시 Desc 힙 채움
  2. 실패 시 Free_Binary_Desc 로 부분 할당분 정리 후 nullptr 반환

  4. Initialize_Prototype 내부에서 모든 데이터가 하위 클래스로 복사되었음이 검증됨 → 임시 Desc 즉시 Free_Binary_Desc
- Magic "SLMD" + version + modelType sanity check (캐스팅 비교로 불일치 파일 거름)

#### **트러블슈팅: DBG_NEW 매크로 충돌 (중요 학습)**

##### 증상
사용자가 Step 1 코드 적용 후 다음 라인에서 컴파일 오류:
```cpp
Mesh.pVertices = ::operator new(static_cast<size_t>(Mesh.iVertexStride) * Mesh.iNumVertices);
```

##### 원인
`Engine_Defines.h:42~49` 에서 CRT 디버그 힙 트래킹을 위해 다음 매크로를 정의:
```cpp
#define _CRTDBG_MAP_ALLOC
#define DBG_NEW new ( _NORMAL_BLOCK , __FILE__ , __LINE__ )
#define new DBG_NEW
```

`#define new DBG_NEW` 는 모든 `new` 토큰을 치환하므로 `::operator new(size)` 가 `::operator new ( _NORMAL_BLOCK , __FILE__ , __LINE__ )(size)` 로 해석되어 syntax error 발생.

##### 해결
`::operator new` 저수준 호출 대신 **평범한 `new T[N]` 형태 사용**. DBG_NEW 매크로는 평범한 new/new[] 와 호환되도록 설계됨.

**Load_Binary_Desc 수정:**
```cpp
// 변경 전 (컴파일 오류):
Mesh.pVertices = ::operator new(static_cast<size_t>(Mesh.iVertexStride) * Mesh.iNumVertices);

// 변경 후:
Mesh.pVertices = new _ubyte[static_cast<size_t>(Mesh.iVertexStride) * Mesh.iNumVertices];
```

**Free_Binary_Desc 수정:**
```cpp
// 변경 전 (컴파일 오류):
::operator delete(Mesh.pVertices);

// 변경 후:
delete[] static_cast<_ubyte*>(Mesh.pVertices);
```

##### 일반 원칙 (CLAUDE.md 에도 반영)
- Engine 코드에서 **`::operator new` / `::operator delete` 같은 저수준 호출 금지**
- 항상 `new T` / `new T[N]` / `delete` / `delete[]` 형태만 사용
- `void*` 에 byte 배열을 할당할 때는 `new _ubyte[size]` + `delete[] static_cast<_ubyte*>(ptr)` 패턴
- `Safe_Delete_Array<T>` 템플릿은 `void*` 에 사용 불가 (컴파일러가 배열 요소 타입을 deduce 못 함)
