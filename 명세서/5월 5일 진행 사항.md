#### `Update_Combat` (Early-Cancel)

```cpp
const _bool bIsAttacking = (BASIC_ATTACK_01/02/03 == eCur);

// (1) OPEN + W → IDLE 캔슬 (단, LMB 동시 입력이면 보류)
if (bIsAttacking && m_bComboWindowOpen
    && Intent.bHasMoveIntent && !Intent.bAttackRequested)
{
    __super::On_ActionFinished();
    Try_Transition(ETOUI(IDLE));
    /* 상태 리셋 */
    return;
}

// (2) LMB 미입력 → 종료
if (!Intent.bAttackRequested) return;

// (3) OPEN window 중 LMB → 다음 액션 즉시 전환
if (bIsAttacking)
{
    if (!m_bComboWindowOpen) return;

    const CHARACTER_ACTION eNext =
        (BASIC_ATTACK_01 == eCur) ? BASIC_ATTACK_02 :
        (BASIC_ATTACK_02 == eCur) ? BASIC_ATTACK_03 :
                                    BASIC_ATTACK_01;   // _03 → _01 루프

    __super::On_ActionFinished();
    if (Try_Transition(ETOUI(eNext))) { /* iComboStep 갱신 */ }
    return;
}

// (4) 비공격 + LMB → _01 진입
Try_Transition(ETOUI(BASIC_ATTACK_01));
```

## 4. 디버깅 여정 (참고용)

### 4-1. _03 OPEN + LMB 시 _03 만 반복되는 현상

증상: LMB 홀드 시 transition 로그가 `13 → 0 → 13 → 0 → 13` 으로 _03 만 반복.

원인: 초기에는 큐잉 방식이었고 bAttackFinished 분기 가드 (`BASIC_ATTACK_03 != eFinished`) 때문에 _03 종료 시 큐가 무시되어 IDLE 로 갔다가 다시 _01 로 돌아가야 하는데, **분기 (4) 에서 _01 진입 직전 어떤 경로로 _03 으로 재진입**.

해결: Early-Cancel 분기 (3) 에서 `_03 == eCur` 가드를 제거하고 **`_03 → _01` 루프** 를 직접 코딩. OnNotify 에는 어떤 콤보 진행 로직도 두지 않아 이중 트리거 차단.

### 4-2. W + LMB 동시 입력 시 콤보 끊김

증상: W 누른 채 LMB 연타하면 OPEN 마다 분기 (1) 의 이동 캔슬이 먼저 발동 → IDLE → _01 무한 반복.

해결: 분기 (1) 가드에 `&& !Intent.bAttackRequested` 추가 — 같은 프레임에 LMB 동시 입력이면 이동 캔슬 보류 → 분기 (3) 의 콤보 진행이 우선 처리.

### 4-3. 단타 시 LMB 미입력 프레임에서 IDLE 캔슬

증상: `Get_MouseBtnState` 단독 사용 시 단타 사이 미입력 프레임에 W 와 겹쳐 분기 (1) 발동.

해결: `Get_MouseBtnDown || Get_MouseBtnState` + Player 측 입력 버퍼 0.18s. Down 엣지로 버퍼 충전, State 로 홀드 시 매 프레임 갱신, 버퍼가 끊김 프레임을 메움.

## 9. 가드 동작 스펙 (확정)

| 입력 상황 | 결과 |
|---|---|
| 비가드 + RMB Hold | `GUARD_START` 진입 |
| `GUARD_START` 자연 종료 + RMB Hold 유지 | `GUARD_LOOP` 자동 전이 |
| `GUARD_START` 자연 종료 + RMB Released | `GUARD_END` 진입 |
| `GUARD_LOOP` 중 + RMB Released | `GUARD_END` 트리거 (LOOP 즉시 캔슬) |
| `GUARD_END` 자연 종료 | `IDLE` 복귀 |

| 가드 중 WASD | Is_GuardLocked 차단으로 LocoMotion 무시 |
| DASH/BACK_DASH 중 RMB Hold | Reject 등록으로 GUARD_START 진입 거부 |
| 가드 중 Space | DASH 진입 허용 (Reject 단방향) |
| 공격 중 RMB Hold | priority 4 > 3 으로 GUARD_START 캔슬 진입 |

핵심 결정: **Held 기반 트리거** (Edge 가 아닌 직접 `bGuardHeld` 검사). RMB input flicker 가 polling 시 1프레임 false 반환하는 케이스 발견 → Edge 검출 신뢰 불가 → Held + 디바운스 0.10s 로 단순화.

4단계 폴백 매칭 + 자동 state 갱신:

3. (any_state, action, weapon) ← state 폴백
4. (any_state, action, DEFAULT)

매칭된 Bind 의 state 로 `m_eCurrentState` 자동 갱신 → CHARACTER_STATE 가 진짜로 의미를 가지게 됨. const 한정자 제거 필요.

## 11. 디버깅 여정 (참고용)

### 11-1. GUARD_START 진입 후 모든 입력 차단
초기 증상: RMB 한 번 누르면 priority 4 stuck → 모든 입력 거부.
원인: GUARD Bind 의 state=GUARD 로 등록했는데 `Body_Player::Find_Bind` 가 m_eCurrentState=LOCOMOTION 와 정확 매칭만 시도. 매칭 실패 → 모션 재생 안 됨 → ACTION_FINISHED 안 옴 → stuck.
해결: `Resolve_ActionKey` 에 4단계 폴백 + state 자동 갱신 도입. CHARACTER_STATE 가 실제 의미를 갖는 시스템으로 재설계.

### 11-2. RMB 한 번 누르면 가드 한 사이클 후 풀림 (= "원샷")
증상: GUARD_START → GUARD_LOOP → GUARD_END 한 사이클 후 IDLE. RMB 계속 누르고 있어도 동일.
진단: Animation tick 로그로 GUARD_LOOP tick 0.47/1.63 에서 갑자기 GUARD_END 로 전환되는 것 확인. LOOP 자체 isLoop=1 정상.
원인: `Get_MouseBtnState` 가 RMB 유지 중에도 1프레임 false 를 반환하는 input flicker. Update_Guard 분기 (2) 가 즉시 GUARD_END 트리거.
해결: Player.cpp 에 `m_fGuardHoldGraceTimer` 디바운스 0.10s 도입. 콤보 LMB 버퍼와 동일 패턴.

### 11-3. 무기 비가시 상태에서 BASIC_ATTACK 진입 시 무기 안 보임 (Step C-6 발견 후속)
원인: `On_Transition` 의 무기 가시성 switch 에 BASIC_ATTACK case 누락 → 이전 이동 상태의 Hide 가 그대로 유지.
해결: BASIC_ATTACK_01/02/03 진입 시 `Set_WeaponsVisible(true)` 추가. GUARD_* 도 동일 처리 (가드 중 무기 노출 유지 결정).
