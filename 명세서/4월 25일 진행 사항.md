## 3. 잔존 이슈 — Dash 시 마우스 Y 둔화 (✅ 4/26 해소 — 단, 원인 진단은 틀렸음)

> **갱신 (2026-04-26)**:

> - K-2 (SPACE 단독 BACK_DASH) 도 체감 무 → §3.3 Ch.6 패치는 불필요
> - 이후 추가 검토에서 **합성 순서 버그** 발견 후 1줄 fix 로 카메라 어색함 자체가 사라짐. [4월 26일 §10](./4월%2026일%20진행%20사항.md) 참조
> - **§3.2 의 "basis Cross degenerate" 추정은 틀린 가설** — Pitch clamp 가 90° 못 미치게 막혀 있어 Cross 가 항상 정상 직교. 참고용으로 남기되 본문 해당 부분은 정정 표기
> - 아래 본문은 당시 분석 기록 보존용

### 3.1 재현

- 플레이어가 카메라 정면을 바라보는 상태 (player forward ↔ camera forward 거의 정반대)
- 그 상태에서 Dash / Back_Dash 발동 → 직후 마우스 Y 입력의 Pitch 응답이 둔해짐
- 마우스 X 한 번 흔들면 정상 복귀

### 3.2 원인 (Codex 검토)

**stale frame + basis 재구성 트리거 부족**:

1. `Player.cpp:71 Query_CameraYaw()` 가 `GameInstance->Get_Transform(D3DTS::VIEW)` 를 읽음.
2. VIEW 행렬은 직전 프레임 `Camera_Follow::Update_PipeLine()` 결과 (`GameInstance.cpp:66` 의 업데이트 순서상).
3. `CCamera_Follow::Late_Update` 에서만 SpringArm 회전 + PipeLine 반영.
4. 결과: **Player::Update 는 이전 프레임 yaw 로 판단**, 카메라는 그 후 새 입력으로 갱신 → 한 프레임 lag.
5. 평소엔 무시 가능. 그러나 "정면 바라보기" 상태에서 각도 차가 π 근처 → 작은 stale 오차도 증폭.
6. Dash 의 루트모션이 즉시 큰 위치 변화를 만들어 mismatch 가 카메라에 강하게 남음.

`Camera_Follow::Apply_SpringArmToTransform` 의 `Cross` 기반 basis 재구성도 일조. SpringArm 의 `q` 전체가 아니라 `look` 만 받아 `right`, `up` 을 다시 짜기 때문에 "look 은 바뀌었는데 basis 가 어색" 한 잠시 상태 발생 가능. Yaw 입력이 들어오면 basis 가 크게 재계산되며 정상화됨 → "X축 이동 시 복귀" 증상의 정체.

> **⚠️ 정정 (2026-04-26)**: 위 "Cross degenerate" 추정은 **틀렸음**. Pitch clamp 가 `[-1.4, 1.0] rad` 으로 90°에 못 미치게 막혀 있어 Cross 결과가 항상 정상 직교 단위벡터로 나옴. 4/26 검증 로깅(`[BasisBroken]`) 0건 출력으로 확인. **실제 원인은 SpringArm 의 Pitch-Yaw 합성 순서** (`XMQuaternionMultiply(qYaw, qPitch)` → math `qPitch * qYaw` → Pitch 가 월드 X 축으로 적용 → Yaw 와 결합되어 Look.y = -sin(Pitch)·cos(Yaw)). [4월 26일 §10](./4월%2026일%20진행%20사항.md) 참조.

### 3.3 패치 방향 (Ch.6 작업)

**A. 카메라 입력 갱신을 Update 단계로 분리**
- 현재: `Late_Update` 에 `[Update_Rotation + Apply_SpringArmToTransform + Update_PipeLine]` 한 번에
- 이후: `Update` 에 `[Update_Rotation]` (yaw/pitch 갱신만) / `Late_Update` 에 `[Apply + PipeLine]`
- → `Player::Update` 가 이번 프레임 카메라 yaw 를 볼 수 있음
- → `Query_CameraYaw` 는 VIEW 행렬 대신 SpringArm 직접 참조

**B. `Apply_SpringArmToTransform` 의 basis 재구성을 q 직접 회전으로 교체**
```cpp
q = LoadFloat4(spring->m_qOrientation);
vRight = XMVector3Rotate((1,0,0,0), q);
vUp    = XMVector3Rotate((0,1,0,0), q);
vLook  = XMVector3Rotate((0,0,1,0), q);
```
3축 모두 같은 q 에서 일관 도출 → "look 만 바뀌고 basis 어색" 상태 자체가 사라짐. (q 는 Yaw + Pitch 만 합성하므로 Roll = 0 보장.)

**검증 절차**:
1. 패치 전 — Player.cpp dash 시작 프레임에 `Query_CameraYaw()` / `m_pSpringArm->Get_Yaw()` / 플레이어 yaw 3개 로깅 → 한 프레임 lag 확인
2. 패치 후 — 같은 로깅으로 lag 사라졌는지 확인 + 재현 시나리오 (정면 바라보기 + Dash) 마우스 Y 응답 즉시성 확인
