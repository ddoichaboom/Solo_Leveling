# Solo_Leveling 개발 진행 가이드

## 개요

CPP_STUDY_3D/Framework의 진행사항을 Solo_Leveling 4-프로젝트 구조에 반영하기 위한 단계별 명세서.

**원본**: `C:\Users\chaho\ddoichaboom\CPP_STUDY_3D\Framework` (2-프로젝트: Engine DLL + Client EXE)
**대상**: `C:\Users\chaho\ddoichaboom\Solo_Leveling` (4-프로젝트: Engine DLL + Client DLL + GameApp EXE + Editor EXE)

### 아키텍처 결정사항

| 항목 | 결정 |
|------|------|
| Transform | abstract CTransform → CTransform_3D / CTransform_2D 분리 |
| Assimp | Editor에만 격리 (바이너리 파이프라인) |
| Client 게임 오브젝트 | CLIENT_DLL export (Editor에서도 사용) |
| Resources 경로 | 기존 Solo_Leveling 구조 유지 (`Resources/ShaderFiles/`, `Resources/Textures/`) |
| 프로젝트 구조 | 4-프로젝트 유지 (Engine DLL + Client DLL + GameApp EXE + Editor EXE) |

### 진행 순서

```
Phase A: PipeLine + Camera          ← View/Proj 행렬 관리 기반
Phase B: Input_Device               ← Raw Input 입력 시스템
Phase C: 정점 포맷 + 셰이더 확장     ← VTXNORTEX, VTXMESH, 라이팅 셰이더
Phase D: Transform 2D/3D + UIObject ← Transform 분리 + UI 베이스
Phase E: VIBuffer_Terrain + Terrain ← 높이맵 지형
Phase F: Light 시스템               ← CLight + CLight_Manager
Phase G: Model/Mesh/Material        ← Editor 바이너리 파이프라인
```

### 복사 불가 — 신규 작성 (Assimp 격리)

Framework에서는 Engine에 Assimp이 직접 들어있어서 복사 불가. 바이너리 전용으로 새로 작성.

| Phase | 파일 | 이유 |
|-------|------|------|
| G | Engine: Model.h/cpp | Framework는 `Importer`, `aiScene*` 사용. 바이너리 `fread` 전용으로 재작성 |
| G | Engine: Mesh.h/cpp | Framework는 `aiMesh*` 파라미터. raw 데이터 기반으로 재작성 |
| G | Engine: Material.h/cpp | Framework는 `aiMaterial*`, `aiTextureType` 사용. 커스텀 구조체 기반으로 재작성 |
| G | Editor: ModelConverter.h/cpp | 완전 신규. Assimp → 바이너리 변환기 |

### Engine_Defines.h 주의

Framework의 Engine_Defines.h에는 Assimp 헤더가 포함되어 있음:
```cpp
// Framework Engine_Defines.h (복사 금지 부분)
#include <assimp/scene.h>
#include <assimp/Importer.hpp>
#include <assimp/postprocess.h>
using namespace Assimp;
```
**Solo_Leveling에 절대 추가하지 않는다.** 나머지 include는 이미 Solo_Leveling에 있으므로 Engine_Defines.h는 복사하지 않고 현행 유지.

## Phase별 빌드 가능 시점 정리

각 Phase 완료 후 빌드 가능한 범위:

| Phase | 빌드 가능 시점 | 비고 |
|-------|--------------|------|
| A | PipeLine + Camera (Engine만) | Camera_Free는 B+D 필요 |
| A+B | Input_Device 포함 빌드 | Camera_Free는 D 필요 |
| A+B+C | 셰이더 추가 빌드 | HLSL 파일 복사만, 아직 사용처 없음 |
| A+B+C+D | **Transform 분리 완료 → Camera_Free 빌드 가능** | 기존 코드 수정 가장 많음 |
| +E | Terrain 추가 | VtxNorTex 셰이더 실사용 시작 |
| +F | Light 추가 | 라이팅 셰이더 실사용 시작 |
| +G | Model 파이프라인 완성 | Monster/ForkLift 빌드 가능 |

**실질적 첫 빌드 체크포인트**: Phase D 완료 후 (Camera_Free가 동작하는 시점).

## 주의사항

### Assimp 격리 원칙 (최우선)

- Engine_Defines.h에 Assimp include **절대 추가 금지**
- Engine/ThirdPartyLib/에 Assimp lib **절대 추가 금지**
- Editor.exe만 Assimp 의존. GameApp/Client는 Assimp 없이 빌드/실행.

### CRT 통일

모든 프로젝트 `/MDd` (Debug) / `/MD` (Release) 동적 CRT 통일.

### CLIENT_DLL export

Framework에는 CLIENT_DLL 매크로가 없음 (Client가 EXE). Solo_Leveling의 Engine_Macro.h에 이미 정의되어 있으므로, 복사한 Client 헤더에 `CLIENT_DLL` 추가만 하면 됨.

### 리소스 경로 통일

Solo_Leveling 기준 상대 경로 (GameApp/Bin/ 또는 VS $(ProjectDir) 기준):
- 셰이더: `../../Resources/ShaderFiles/...`
- 텍스처: `../../Resources/Textures/...`
- 모델: `../../Resources/Models/...`

### Transform 캐스팅

3D 전용 메서드 호출 시 `static_cast<CTransform_3D*>(m_pTransformCom)`. Framework의 Camera_Free.cpp에 이미 이 패턴이 사용됨.

### 향후 확장 (이 문서 범위 밖)

- 스켈레탈 애니메이션 (Bone, Channel, KeyFrame → 바이너리 포맷 확장)
- 네비게이션 메시 (Cell 기반)
- Collider (AABB, OBB, Sphere)
- 인스턴싱, 그림자, 포스트 프로세싱
