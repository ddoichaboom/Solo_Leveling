# 그림자 (Shadow)

## 깊이 버퍼 그림자

- 해당 예제를 후처리 셰이더에 적용할 수 있게 학습

- 특정 픽셀을 표현하고자 할 때 특정 광원 기준으로 사이에 객체가 있는지 여부를 알아야 함.
- 광원 기준으로 표현된 깊이를 비교해야 한다.
- 광원이 바라본 장면의 픽셀들의 깊이를 보관해 놓아야 한다. -> 렌더 타겟에 담는다
- 광원을 마치 카메라 처럼 생각하면 될듯
- 2 pass Rendering을 수행하게된다
- 움직임이 없고 애니메이션이 없는 애들은 로딩 때 드로우 콜을 한번만 수행해서 저장시켜놓는다.
- Light_Manager에 등록하기에는 흐름이 안맞아서 Shadow라는 클래스를 따로 구현해놓는다.
- 플레이어의 그림자를 표현하기 위해서는 플레이어를 렌더 그룹에 2번 추가해야함
- Render_Shadow 함수를 GameObject에 선언
- Diffuse를 바인딩해줘야 알파값을 통해 퀄리티 좋은 그림자를 만들 수 있다.
- 장치가 초기화 시에 사용하던 깊이 버퍼에 그대로 수행했기 때문에 화면 상에 표시됨 (깊이 테스트 함께 수행되버림)
- 그림자 용도로 깊이 버퍼 하나 더 만들 예정
- 렌더 타겟에 그려진 광원 기준 장면과 실제 내가 보고 있는 화면의 사이즈가 차이나서 그림자의 퀄리티가 낮아짐
- LightDepthRenderTarget의 크기를 키워서 판단 -> 해상도를 아예 키워서 ( CasCade 그림자 기법)
- 사이즈가 달라지면 ViewPort 사이즈, 깊이 버퍼 모두 변경되어야 함.
