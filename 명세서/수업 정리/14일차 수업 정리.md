# Channel

- 키 프레임 사이 사이 에서는 값을 보간하여 처리해준다.
- 마지막 키 프레임 넘어갔을 때는 마지막 상태값을 그대로 남은 Duration동안 유지

## 보간처리

- 현재 프레임 기준 왼쪽 프레임, 오른쪽 프레임의 Scale이 필요
- 내 현재 위치 왼쪽 프레임의 인덱스를 저장
- vScale, vTranslation은 XMVectorLerp 함수를 사용하면 됨
- vRotation은 XMQuaternionSlerp함수 사용

# Animation

- 강사님 코드에서는 vector로 Animation을 담고 인덱스로 관리하는데, 만약 애니메이션 이름으로 관리하고 싶다면 map으로 사용
- AIAnimation에 szName이 존재, 해당 이름으로 등록해도 됨

## 애니메이션 별도 분기 처리

### Animation Loop

- 애니메이션이 루프인지 bool 변수로 파라미터 전달
- Updat_TransformMatrices 함수 내부에서 애니메이션이 끝났는지 여부도 반환
- 값이 0으로 던져지면 애니메이션이 끝나고 루프가 시작된 것으로 판단(이미 한번 끝난 것으로 판단) 하여 내 왼쪽 프레임 위치도 0으로 초기화
- 이유 : 값을 던질 때 더하면서 던져서 0을 던지는 경우는 유일한 경우로 판단 가능함.

# 현재 구조의 문제점

1. Bone 얕은 복사로 인한 공유로 인해 같은 애니메이션이 그려짐
2. Animation 얕은 복사로 인한 애니메이션의 속도 증가 (속도 제어 문제 발생)
3. Channel도 공유되고 있음
4. 한 애니메이션에서의 선형 보간은 했지만 애니메이션 전환 사이에서 선형 보간은 구현되지 않았다.
5. 애니메이션 재생할 때 제자리에서 애니메이션 재생은 로컬에서 뼈를 움직이면서 진행하지만, 움직이면서 애니메이션을 재생하면 아직 월드 공간으로의 이동이 반영되지 않았기 때문에 애니메이션 재생시에 순간이동 되는 현상이 나타남.
