# Navigation Mesh

- 플레이어나 몬스터가 갈 수 있는 길을 Mesh로 깔아둔다.
- 충돌로 움직임을 제어하는 것은 모든 충돌을 검사해야하기 때문에 부담스러운 부분이 있다.
- 그래서 길 위에서만 움직인다라고 하면 길 끝부분에서는 충돌이 일어나서 못움직이는 것처럼 움직이지 못한다.

- 매 프레임당 Navigation Mesh의 삼각형을 나갔는지 아닌지만 판단하면 됨

## 셀(삼각형)

- 삼각형의 3개의 선분과 판단을 어떻게 할 것인가?
- (1) 결과 위치를 미리 계산.
- (2) 선분위의 점과 결과 위치를 빼서 방향벡터를 구함
- (3) 선분의 수직 벡터를 구함
- (4) 두 방향벡터를 정규화한 후 내적함
- (5) 내적 결과가 양수면 (0 ~ 90 / 270 ~ 360) 삼각형을 나갔다라고 판단
- (6) 내적 결과가 음수면 (90 ~ 270) 삼각형 안쪽이라고 판단

## Editor Tool의 구현

- 삼각형 각 선분 위치의 차 (방향벡터) -> 시계방향으로 나오게끔 점 순서 잡을것
- (아이디어)시계 방향/ 반 시계 방향 배치 된 삼각형 컬러 다르게 표현하기
- 찍는 순서대로 말고 3개를 찍으면 3개의 외적을 가지고 어떤 순서대로 넣어야 시계방향이 되는지 판단해서 저장
- 스냅 기능을 넣는게 나을듯 ( 이미 있는 점 부근에 Picking을 하면 해당 점을 공유하는 것으로 판단 )
