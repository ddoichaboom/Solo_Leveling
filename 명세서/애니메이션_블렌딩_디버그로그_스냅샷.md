# 애니메이션 블렌딩 디버그 로그 스냅샷

cross-fade 블렌딩 / Root Motion 추적 디버깅용 `OutputDebugStringA` 코드 모음. 비슷한 증상(메쉬 점프, backward, blend 끊김 등) 발생 시 아래 4 개를 같은 위치에 다시 끼워넣고 Visual Studio Output 창에서 흐름을 확인.

## 사용법

1. 아래 4 개 블록을 각자 지정된 위치에 임시로 추가
2. 빌드 후 GameApp 실행 → 재현 시나리오 수행 → Output 창의 `[BlendStart]` / `[InstantSwitch]` / `[Regular RM][...]` / `[Blend RM]` 라인 수집
3. 분석이 끝나면 모두 제거

## prefix 의미

| prefix | 의미 |
|---|---|
| `[BlendStart]` | 블렌드 진입 이벤트. From/To 이름 + RM 플래그 + BlendTime + 이전에 블렌드 중이었는지 |
| `[InstantSwitch]` | 블렌드 안 거치고 즉시 전환 |
| `[Regular RM][INIT]` | RM 클립 첫 프레임에서 bind 캡처 (delta=0) |
| `[Regular RM][TICK]` | 매 프레임 정상 RM 추출 (T_cur, localDelta, finalDelta) |
| `[Regular RM][WRAP]` | 루프 wrap (delta 0 처리) |
| `[Regular RM][DISABLED/NO_RM_CLIP/...]` | 어떤 사유로 RM 비활성됐는지 |
| `[Blend RM]` | 블렌드 중 매 프레임. From/To delta, weight, lerp 결과, static From 여부, init 플래그 |

## 3) 정규 Extract_RootMotion — 매 프레임 상태 (early-return 포함)

`Engine/Private/Model.cpp` `Extract_RootMotion` 의 함수 body 전체를 아래로 교체.

```cpp
void CModel::Extract_RootMotion(_float fPrevTrackPos)
{
    auto LogExit = [&](const _char* pTag) {
        const _char* pName = (m_iCurrentAnimationIndex < m_iNumAnimations)
            ? m_Animations[m_iCurrentAnimationIndex]->Get_Name() : "(none)";
        char buf[256];
        sprintf_s(buf, "[Regular RM][%s] anim=%s rmEnabled=%d delta=(%.4f,%.4f,%.4f)\n",
            pTag, pName, m_bRootMotionEnabled ? 1 : 0,
            m_vLastRootMotionDelta.x, m_vLastRootMotionDelta.y, m_vLastRootMotionDelta.z);
        OutputDebugStringA(buf);
    };

    if (!m_bRootMotionEnabled || m_iRootBoneIndex < 0)
    {
        m_vLastRootMotionDelta = {};
        LogExit("DISABLED");
        return;
    }

    if (static_cast<_uint>(m_iRootBoneIndex) >= m_iNumBones)
    {
        m_vLastRootMotionDelta = {};
        LogExit("BAD_ROOT");
        return;
    }

    if (m_iCurrentAnimationIndex >= m_iNumAnimations ||
        !m_Animations[m_iCurrentAnimationIndex]->Get_UseRootMotion())
    {
        m_vLastRootMotionDelta = {};
        m_bRootMotionInitialized = false;
        LogExit("NO_RM_CLIP");
        return;
    }

    CBone* pRootBone = m_Bones[m_iRootBoneIndex];
    if (nullptr == pRootBone)
    {
        m_vLastRootMotionDelta = {};
        LogExit("NULL_BONE");
        return;
    }

    const _float4x4* pMatrix = pRootBone->Get_TransformationMatrixPtr();
    _float3 T_cur = { pMatrix->_41, pMatrix->_42, pMatrix->_43 };

    _float fCurrTrackPos = m_Animations[m_iCurrentAnimationIndex]->Get_CurrentTrackPosition();
    _bool bWrapped = (fCurrTrackPos < fPrevTrackPos);

    if (!m_bRootMotionInitialized)
    {
        m_vBindRootTranslation = T_cur;
        m_vPrevRootTranslation = T_cur;
        m_vLastRootMotionDelta = {};
        m_bRootMotionInitialized = true;

        char buf[256];
        sprintf_s(buf, "[Regular RM][INIT] T_cur=(%.4f,%.4f,%.4f) bind set\n",
            T_cur.x, T_cur.y, T_cur.z);
        OutputDebugStringA(buf);
    }
    else if (bWrapped)
    {
        m_vLastRootMotionDelta = {};
        m_vPrevRootTranslation = T_cur;

        char buf[256];
        sprintf_s(buf, "[Regular RM][WRAP] T_cur=(%.4f,%.4f,%.4f) prevTrack=%.3f curTrack=%.3f\n",
            T_cur.x, T_cur.y, T_cur.z, fPrevTrackPos, fCurrTrackPos);
        OutputDebugStringA(buf);
    }
    else
    {
        _float3 vLocalDelta{};
        vLocalDelta.x = T_cur.x - m_vPrevRootTranslation.x;
        vLocalDelta.y = T_cur.y - m_vPrevRootTranslation.y;
        vLocalDelta.z = T_cur.z - m_vPrevRootTranslation.z;

        m_vPrevRootTranslation = T_cur;

        _matrix matPre = XMLoadFloat4x4(&m_PreTransformMatrix);
        matPre.r[0] = XMVectorSetW(XMVector3Normalize(matPre.r[0]), 0.f);
        matPre.r[1] = XMVectorSetW(XMVector3Normalize(matPre.r[1]), 0.f);
        matPre.r[2] = XMVectorSetW(XMVector3Normalize(matPre.r[2]), 0.f);
        matPre.r[3] = XMVectorSet(0.f, 0.f, 0.f, 1.f);

        _vector vDelta = XMVectorSet(vLocalDelta.x, vLocalDelta.y, vLocalDelta.z, 0.f);
        vDelta = XMVector3TransformNormal(vDelta, matPre);
        XMStoreFloat3(&m_vLastRootMotionDelta, vDelta);

        const _char* pName = m_Animations[m_iCurrentAnimationIndex]->Get_Name();
        char buf[384];
        sprintf_s(buf, "[Regular RM][TICK] anim=%s T_cur=(%.4f,%.4f,%.4f) localDelta=(%.4f,%.4f,%.4f) finalDelta=(%.4f,%.4f,%.4f)\n",
            pName, T_cur.x, T_cur.y, T_cur.z,
            vLocalDelta.x, vLocalDelta.y, vLocalDelta.z,
            m_vLastRootMotionDelta.x, m_vLastRootMotionDelta.y, m_vLastRootMotionDelta.z);
        OutputDebugStringA(buf);
    }

    _float4x4 matFixed = *pMatrix;
    matFixed._41 = m_vBindRootTranslation.x;
    matFixed._42 = m_vBindRootTranslation.y;
    matFixed._43 = m_vBindRootTranslation.z;
    pRootBone->Set_TransformationMatrix(XMLoadFloat4x4(&matFixed));
}
```

## 4) Extract_RootMotion_Blended — 블렌드 중 매 프레임 상태

`Engine/Private/Model.cpp` `Extract_RootMotion_Blended` 의 마지막 root 본 bind-fix 직전(또는 직후)에 추가. 변수명 `vDeltaFrom`, `vDeltaTo`, `vLerped`, `fWeight` 가 함수 내부 변수.

```cpp
{
    const _char* pFromName = m_Animations[m_iBlendFromIndex]->Get_Name();
    const _char* pToName = m_Animations[m_iBlendToIndex]->Get_Name();
    _bool bFromRM = m_Animations[m_iBlendFromIndex]->Get_UseRootMotion();
    _bool bToRM = m_Animations[m_iBlendToIndex]->Get_UseRootMotion();
    _bool bHasFR = (0 != m_bHasPoseFrom[(m_iRootBoneIndex >= 0) ? m_iRootBoneIndex : 0]);
    _bool bHasTR = (0 != m_bHasPoseTo[(m_iRootBoneIndex >= 0) ? m_iRootBoneIndex : 0]);
    char buf[512];
    sprintf_s(buf,
        "[Blend RM] %s(RM=%d,hasRoot=%d) -> %s(RM=%d,hasRoot=%d) "
        "w=%.3f static=%d initF=%d initT=%d "
        "DeltaFrom=(%.4f,%.4f,%.4f) DeltaTo=(%.4f,%.4f,%.4f) "
        "Lerped=(%.4f,%.4f,%.4f) Final=(%.4f,%.4f,%.4f)\n",
        pFromName, bFromRM ? 1 : 0, bHasFR ? 1 : 0,
        pToName, bToRM ? 1 : 0, bHasTR ? 1 : 0,
        fWeight, m_bFromIsStatic ? 1 : 0,
        m_bBlendRMInitialized_From ? 1 : 0, m_bBlendRMInitialized_To ? 1 : 0,
        vDeltaFrom.x, vDeltaFrom.y, vDeltaFrom.z,
        vDeltaTo.x, vDeltaTo.y, vDeltaTo.z,
        vLerped.x, vLerped.y, vLerped.z,
        m_vLastRootMotionDelta.x, m_vLastRootMotionDelta.y, m_vLastRootMotionDelta.z);
    OutputDebugStringA(buf);
}
```

## 분석 시 체크 포인트

- **`[BlendStart]` 의 RM 플래그**: From / To 각각 RM 사용 여부 — RM 비대칭 전환은 visual 점프나 motion 손실을 유발 가능
- **`[Blend RM]` 의 DeltaFrom / DeltaTo 부호**: PreTransform 적용 전 애니메이션 공간 값. 음수가 보이면 해당 클립 자체가 root 를 그쪽으로 움직이는 것
- **`[Blend RM]` 의 Final 부호**: PreTransform 적용 후 character world delta. 의도와 반대 부호면 PreTransform / 좌표계 의심
- **`[Regular RM][INIT]` 직후 `[Regular RM][TICK]` 첫 줄의 localDelta**: 큰 값이면 blend → 정규 전환 시점에 prev 가 stale 해서 점프. blend 종료 시 prev 핸드오버 필요
- **TICK 의 localDelta 가 부호가 바뀌는 구간**: 애니메이션 자체가 settle / recoil 모션을 root 에 넣어둔 것. RM 으로 적용하지 말지 결정 필요 (예: Run_End 류)
- **`[Regular RM][DISABLED/NO_RM_CLIP]` 가 예상 외로 자주 등장**: RM 클립이라 생각했는데 SLMD 의 UseRootMotion 플래그가 false 인 경우. Editor 변환 단계 데이터 의심

## 과거 사례 (2026-04-28)

LOCOMOTION 전체 사이클 디버깅 결과:

- **확정된 RM 플래그**:
  - RM=0: Idle, Run, Run_Fast, Run_FastLeft, Run_FastRight
  - RM=1: Dash, BackDash, Run_End, Run_End_Left, Run_End_Right

- **발견된 이슈**:
  1. **Run_End_Left/Right 가 종료 직전에 root 를 +방향으로 당김** → final delta 음수 → character backward 0.04~0.06m. 해법: 해당 클립의 UseRootMotion=false 처리 (런타임에서 `Set_AnimationUseRootMotion`)
  2. **DASH 진입 시 0.25m forward jump**: Blend 종료 후 정규 path 첫 프레임이 INIT 으로 prev 캡처 → 다음 프레임에서 큰 delta. 해법: blend 종료 시 `m_vPrevRootTranslation_To` 를 정규 RM 상태로 핸드오버
