- **설계 결정**: 파생 클래스 소유 → CGameObject 공통 소유로 변경 (대안 A)

#### Shadowing 문제 분석 및 해결
- **문제**: CTerrain/CBackGround가 각자 `m_pVIBufferCom`을 private 멤버로 소유 중. CGameObject에 동명 멤버 추가 시 파생 것이 베이스를 가림 → 베이스 getter가 nullptr 반환
- **대안 비교**:
  - A. **베이스로 올리기** (채택) — 파생에서 제거, m_pTransformCom 패턴과 동일
  - B. 다른 이름으로 추가 — 같은 객체 3곳 참조 (RefCount 3), 관리 복잡
  - C. map에서 검색 — 코드 변경 없으나 매 피킹마다 map 순회 + dynamic_cast 비용
- **해결**: 대안 A — `CVIBuffer* m_pVIBufferCom`을 CGameObject protected로 이동, 파생에서 동명 멤버 삭제

#### RefCount 패턴 정리
- **이중 참조 구조**: m_Components map (소유자 1) + 멤버 포인터 (소유자 2) = RefCount 2
- **Add_Component 내부**에서 Safe_AddRef 수행 → 파생 클래스에서 별도 AddRef 불필요
- **Release 순서**: 파생 Free() → 자기 멤버만 Release / 베이스 Free() → map 순회 Release + 공통 멤버 Release
- **파생 전용 메서드 필요 시**: static_cast 사용 (기존 m_pTransformCom 선례: BackGround.cpp CTransform_2D 캐스팅)

## 트러블슈팅

### 1. VIBuffer_Terrain PICK_DATA 할당 순서 버그
- **증상**: CVIBuffer_Terrain::Initialize_Prototype() 줄 161에서 크래시
- **원인**: PICK_DATA의 pVerticesPos에 쓰기 전에 메모리 할당이 누락 (할당 코드가 반복문 뒤에 위치)
- **해결**: `new _float3[]` 할당을 for 루프 이전으로 이동

### 2. PipeLine 함수명 오타
- **증상**: LNK2019 unresolved external
- **원인**: `Comput_WorldRay` → `Compute_WorldRay` (e 누락)
- **해결**: 함수명 수정

### 3. Compute_WorldRay Y축 공식 부호 오류
- **증상**: Terrain 클릭 시 Pick_Object 진입은 되지만 pPicked가 항상 nullptr
- **디버깅**: 하드코딩 Ray (64,100,64)→(0,-1,0) 테스트 시 피킹 성공 → Compute_WorldRay가 원인
- **원인**: Y 공식 `(-2.f * fViewportY / fViewportHeight - 1.f)` → `+ 1.f`이어야 함
- **해결**: 부호 수정 (`-1.f` → `+1.f`)

### 4. Clear_Selection 댕글링 포인터 크래시
- **증상**: 빈 공간 클릭 시 0xFFFFFFFFFFFFFFFF 접근 위반
- **원인**: Safe_Release()는 RefCount가 0일 때만 포인터를 nullptr로 설정. 선택된 오브젝트의 RefCount가 2(Layer+Panel_Manager)이므로 Release 후 1→포인터 유지→다음 프레임에서 댕글링
- **핵심 발견**: `Safe_Release`는 RefCount > 0이면 포인터를 nullptr로 만들지 않음
- **해결**: `Clear_Selection()`에서 `Safe_Release` 후 `m_pSelectedObject = nullptr;` 추가

### 5. LNK2019 CLevel_GamePlay::Create
- **증상**: Editor 빌드 시 CLevel_GamePlay::Create 미해결 외부 참조
- **원인**: Client의 게임 레벨 클래스에 CLIENT_DLL export 없음
- **결정**: Client 게임 로직 export 대신 Editor 전용 CLevel_Editor 빈 셸 생성
- **이유**: 게임 로직 자체를 export하면 의존성이 커지고, Editor는 자체 레벨 관리가 필요

### 목차 5: Engine 클래스 설계 — CModel, CMesh, CMaterial, CBone, CAnimation, CChannel

#### 전체 소유 구조
```
CModel (CComponent 상속)
├── vector<CMesh*>        — 메시 배열 (지오메트리)
├── vector<CMaterial*>    — 머터리얼 배열 (텍스처)
├── vector<CBone*>        — 본 배열 (스켈레톤 계층)
└── vector<CAnimation*>  — 애니메이션 클립 배열
         └── vector<CChannel*>  — 채널 배열 (본별 키프레임)
```

#### CModel — 모델 전체 관리자
- **상속**: CComponent (Prototype/Clone 패턴)
- **핵심 멤버**: m_eType(MODEL), m_Meshes, m_Materials, m_Bones, m_Animations, m_iCurrentAnimationIndex, m_PreTransformMatrix
- **Initialize 호출 순서**: ① Ready_Bones → ② Ready_Meshes → ③ Ready_Materials → ④ Ready_Animations
  - 본이 먼저 만들어져야 Mesh/Animation이 Get_BoneIndex()로 본 검색 가능
- **Clone 전략**: Mesh/Material → 공유(AddRef), Bone/Animation → 깊은 복사 (인스턴스별 독립 포즈/재생)

#### CMesh — 메시 (지오메트리)
- **상속**: CVIBuffer (VB/IB/DrawIndexed)
- **핵심 멤버**: m_szName, m_iMaterialIndex, m_BoneIndices(글로벌 인덱스), m_OffsetMatrices, m_BoneMatrices[256]
- **NonAnim**: Ready_NonAnimMesh → VTXMESH 정점, PreTransformMatrix 직접 적용
- **Anim**: Ready_AnimMesh → VTXANIMMESH 정점, BlendIndex/Weight 채움, OffsetMatrix 저장
- **Bind_BoneMatrices 공식**: BoneMatrix[i] = OffsetMatrix[i] × Bones[BoneIndex[i]].CombinedMatrix

#### CMaterial — 머터리얼 (텍스처 관리)
- **상속**: CBase (CComponent 아님)
- **구조**: `vector<SRV*> m_Materials[AI_TEXTURE_TYPE_MAX]` — 2차원 배열 [텍스처타입][인덱스]
- Initialize: aiMaterial에서 파일명 추출 → 경로 조합 → DDS/WIC로 SRV 생성

#### CBone — 본 (스켈레톤 노드)
- **상속**: CBase
- **멤버 4개**: m_szName, m_iParentIndex(-1이면 루트), m_TransformationMatrix(로컬), m_CombinedTransformationMatrix(월드)
- **평탄화**: aiNode 트리를 DFS로 순회 → 배열로 변환. 부모 인덱스 < 자식 인덱스 보장
  - 재귀 없이 단순 for 루프로 전체 계층 갱신 가능
- **계산**: Combined = Local × Parent.Combined

#### CAnimation — 애니메이션 클립
- **상속**: CBase
- **핵심 멤버**: m_fDuration, m_fTickPerSecond, m_fCurrentTrackPosition, m_Channels, m_CurrentKeyFrameIndices
- **m_CurrentKeyFrameIndices가 Animation에 있는 이유**: Channel 공유 가능하지만 재생 위치는 인스턴스별 독립 필요
- **재생 로직**: TrackPosition += TicksPerSecond × DeltaTime → Duration 초과 시 루프/종료 → 각 Channel에 보간 지시

#### CChannel — 채널 (본 1개의 키프레임)
- **상속**: CBase
- **핵심 멤버**: m_iBoneIndex, m_KeyFrames(vector<KEYFRAME>)
- **Assimp 키프레임 통합**: Scale/Rotation/Position 별도 배열 → 하나의 KEYFRAME 배열로 통합 (max 크기)
- **보간**: 브래킷 탐색 → ratio 계산 → Scale/Position은 Lerp, Rotation은 Slerp → XMMatrixAffineTransformation → 본에 적용

#### 전체 프레임 흐름
```
① CAnimation: TrackPosition 전진 → 각 Channel 보간 → CBone.LocalMatrix 갱신
② CBone: Root부터 순회 → CombinedMatrix = Local × Parent.Combined
③ CMesh: BoneMatrix = Offset × Combined → 셰이더 전송 → Material 바인딩 → DrawIndexed
```
